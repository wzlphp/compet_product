# 亚马逊竞品分析需求文档 v3.0

> 参考卖家精灵(SellerSprite)竞品查询功能优化

## 项目背景

作为亚马逊运营，分析竞争链接的核心是**找到对方的核心优势、运营漏洞和流量逻辑**，从而反向优化自身链接、制定差异化策略。

核心关注维度：**基础信息、流量转化、产品运营、合规风险**四大类。

---

## 一、基础信息：锚定竞品核心定位

### 1.1 ASIN 基础数据

| 数据项 | 说明 | 工具参考 |
|--------|------|----------|
| **ASIN** | 产品唯一标识 | 直接抓取 |
| **类目节点** | BSR 所属类目（大类+子类） | Keepa / 卖家精灵 |
| **上架时间** | 判断产品生命周期阶段 | Keepa |
| **品牌** | 品牌名称及是否备案 | Amazon 前台 |
| **卖家类型** | FBA / FBM / Amazon自营 | Amazon 前台 |
| **变体数量** | 父子ASIN关系 | Amazon API |

### 1.2 价格体系（含历史趋势）

| 数据项 | 说明 | 数据源 |
|--------|------|--------|
| **当前售价** | 实时价格 | Amazon 前台 |
| **历史价格曲线** | 价格波动趋势 | Keepa / CamelCamelCamel |
| **促销价** | LD/BD/优惠券价格 | Amazon 前台 |
| **变体价差** | 不同变体定价策略 | 批量抓取 |
| **竞品价格带** | 同类目价格分布 | 卖家精灵市场分析 |

### 1.3 销量与排名

| 数据项 | 说明 | 计算方式 |
|--------|------|----------|
| **BSR 排名** | 大类目 + 子类目排名 | Amazon 前台 |
| **BSR 历史曲线** | 排名波动趋势 | Keepa |
| **预估月销量** | 基于 BSR 反推 | 卖家精灵算法 |
| **预估月销售额** | 销量 × 价格 | 计算得出 |
| **各变体销量占比** | 判断热销款 | Review增量反推 |

## 二、流量与转化：拆解获客逻辑

### 2.1 关键词流量（核心功能）

#### ASIN 反查关键词

| 数据项 | 说明 | 参考工具 |
|--------|------|----------|
| **自然流量词** | 自然搜索排名关键词 | 卖家精灵/Helium10 |
| **广告流量词** | SP/SB 广告投放词 | 卖家精灵广告洞察 |
| **关键词排名** | 首页/前3页/前10 | 实时监控 |
| **搜索量** | 月均搜索量 | 卖家精灵/Merchant Words |
| **点击集中度** | 前3 ASIN 点击占比 | 卖家精灵 |
| **转化率** | 关键词转化率预估 | ABA数据 |

#### 关键词矩阵

```
┌─────────────────────────────────────────────────────────┐
│  高搜索量 + 低竞争 = 🎯 核心机会词                        │
│  高搜索量 + 高竞争 = ⚔️ 主战场词（需要广告）              │
│  低搜索量 + 低竞争 = 🔍 长尾词（精准转化）                │
│  低搜索量 + 高竞争 = ❌ 放弃词                           │
└─────────────────────────────────────────────────────────┘
```

### 2.2 广告分析

| 数据项 | 说明 |
|--------|------|
| **广告位分布** | 搜索顶部/商品页面/搜索其他位置 |
| **广告词数量** | 投放关键词总数 |
| **广告占比** | 广告流量 vs 自然流量 |
| **广告趋势** | 近30天广告投放变化 |
| **竞价预估** | 建议竞价范围 |

### 2.3 活动与促销

| 数据项 | 说明 |
|--------|------|
| **当前促销** | LD/BD/7天秒杀/优惠券/满减 |
| **促销历史** | 历史促销频率和力度 |
| **活动日历** | 促销时间规律 |
| **Coupon 金额** | 优惠券面值和门槛 |

## 三、产品与评价：转化核心要素

### 3.1 Review 深度分析（重要！）

| 分析维度 | 具体内容 |
|----------|----------|
| **评分分布** | 5星/4星/3星/2星/1星占比 |
| **评价总数** | 累计评价数 |
| **评价增速** | 日均/周均增评数 |
| **VP 占比** | Verified Purchase 比例 |
| **评价趋势** | 近30/60/90天评价曲线 |

#### 评价内容挖掘

| 分析项 | 价值 |
|--------|------|
| **好评高频词** | 用户认可的卖点 → 可复制到自身 Listing |
| **差评高频词** | 用户痛点 → **差异化机会点** |
| **竞品未满足需求** | 差评中反复出现的期望 → 产品改进方向 |
| **情感分析** | 正面/负面/中性评价比例 |

### 3.2 Listing 质量评估

| 评估项 | 评分标准 |
|--------|----------|
| **标题优化度** | 关键词覆盖、字符长度、可读性 |
| **主图质量** | 白底、清晰度、角度 |
| **副图完整度** | 场景图、细节图、尺寸图、信息图 |
| **视频** | 是否有视频、视频质量 |
| **五点描述** | 卖点清晰度、关键词嵌入 |
| **A+ 页面** | 是否有 A+、设计质量 |
| **品牌故事** | 是否启用 Brand Story |

### 3.3 QA 分析

| 数据项 | 说明 |
|--------|------|
| **QA 数量** | 问答总数 |
| **高频问题** | 用户最关心的问题 |
| **卖家回复率** | 是否积极回复 |
| **问题类型分布** | 功能/尺寸/兼容性/使用方法 |

## 四、竞品监控：持续追踪

### 4.1 监控维度

| 监控项 | 频率 | 报警条件 |
|--------|------|----------|
| **价格变动** | 每日 | 变动 >5% |
| **BSR 变动** | 每日 | 排名变化 >20% |
| **库存状态** | 每日 | 断货/补货 |
| **评价变化** | 每日 | 单日增评 >10 |
| **关键词排名** | 每周 | 首页掉出 |
| **新增变体** | 每周 | 新 ASIN 上架 |
| **Listing 变更** | 每周 | 标题/图片/价格修改 |


## 五、产品与运营策略

### 5.1 供应链分析

| 维度 | 关注点 |
|------|--------|
| **发货方式** | FBA/FBM/SFP |
| **库存深度** | 库存天数预估 |
| **断货历史** | 是否有断货记录 |
| **物流时效** | Prime 配送时效 |

### 5.2 运营策略反推

| 分析项 | 判断依据 |
|--------|----------|
| **新品推广期** | 评价增速 + 广告力度 |
| **成熟期维护** | 稳定出单 + 低广告占比 |
| **清仓期** | 大幅降价 + BSR 波动 |
| **测评痕迹** | 评价增速异常、VP 比例低 |

### 5.3 品牌运营

| 分析项 | 内容 |
|--------|------|
| **品牌旗舰店** | 是否有 Store、设计质量 |
| **产品线布局** | SKU 数量、品类覆盖 |
| **站外引流** | 社交媒体、独立站、红人合作 |

## 六、合规与风险

| 风险类型 | 关注点 | 反制策略 |
|----------|--------|----------|
| **差评硬伤** | 产品设计/质量缺陷 | 针对性优化，作为自身卖点 |
| **侵权风险** | 专利/商标/版权问题 | 规避或获取授权 |
| **违规操作** | 刷评/刷单/操纵排名 | 不跟进，等待其被惩罚 |
| **断货风险** | 供应链不稳定 | 趁机抢占流量 |

## 七、分析方法论

### 7.1 竞品分层

| 层级 | 定义 | 分析重点 |
|------|------|----------|
| **头部竞品** | BSR Top 10 | 流量策略、运营节奏 |
| **对标竞品** | 排名相近 | 价格、转化、差异化 |
| **潜力竞品** | 新品快速增长 | 差异化卖点、推广策略 |
| **长尾竞品** | 小众但稳定 | 细分需求、精准定位 |

### 7.2 核心公式

```
自身优化方向 = 复制竞品有效策略 
             + 规避竞品漏洞 
             + 放大自身差异化优势
```

### 7.3 定期复盘

| 频率 | 内容 |
|------|------|
| **每日** | 价格、BSR、库存快速扫描 |
| **每周** | 关键词排名、评价变化分析 |
| **每月** | 完整竞品报告、策略调整 |
| **季度** | 市场格局、新入场者分析 |

## 输出要求

### 阶段一：运营分析报告 (claude2)
- 竞品基础信息卡片
- 关键词矩阵（含流量词/广告词）

- 数据可视化看板设计
- 竞品监控报警流程

- 数据采集模块（Amazon/Keepa API）
- 关键词分析引擎
- 可视化报告生成器
- 竞品监控定时任务
